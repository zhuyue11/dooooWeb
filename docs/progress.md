# dooooWeb Implementation Progress

Tracks what has been completed, what's in progress, and what's remaining across all 7 phases to reach full dooooApp feature parity.

Last updated: 2026-04-26

## Tech Stack

- **Build**: Vite + React 18 + TypeScript
- **Routing**: React Router v7 (`createBrowserRouter`)
- **Styling**: Tailwind CSS v4 (`@tailwindcss/vite`, CSS custom properties for theming)
- **State**: React Query (TanStack Query) + Context API (5 providers: auth, theme, language, display, websocket)
- **i18n**: i18next + react-i18next (19 languages, RTL support for Arabic/Farsi)
- **HTTP**: Axios with token interceptor + 401 redirect
- **Real-time**: WebSocket context with exponential backoff reconnect
- **E2E**: Playwright (22 spec files + fixtures + helpers + seed data)

## Architecture Summary

- **Entry**: `src/App.tsx` ŌĆö provider tree: QueryClient ŌåÆ Language ŌåÆ Theme ŌåÆ Display ŌåÆ Auth ŌåÆ WebSocket ŌåÆ RouterProvider
- **Routing**: `src/router.tsx` ŌĆö `PublicRoute` (login/register/forgot) + `ProtectedRoute` ŌåÆ `AppShell` (sidebar + header + `<Outlet>`)
- **API layer**: `src/lib/api/index.ts` ŌĆö 50+ typed functions covering auth, tasks, task instances, categories, events, event instances, groups, targets, plans, notifications, search
- **Contexts**: `src/lib/contexts/` ŌĆö auth-context, theme-context, language-context, display-context, websocket-context
- **Types**: `src/types/api.ts` (Task, Event, Group, Notification), `src/types/target.ts` (Target, Plan, Template, Execution)
- **Calendar**: `src/components/calendar/` ŌĆö MonthGrid, WeekGrid, DayTimeline, ItemSidePanel, ItemFormModal, RecurringScopeModal, CalendarHeader, ItemRow, ItemCard, ItemPanel
- **UI primitives**: `src/components/ui/` ŌĆö Button, Card, Input, Icon, CalendarPopover, RepeatPopover, TimePicker, DurationPicker, ReminderPicker, TimeOfDayPicker, TimeZonePicker
- **Layout**: `src/components/layout/` ŌĆö AppShell (detects group routes via `useMatch`, passes `groupContext` to Sidebar), Sidebar (8 main nav items, collapsible; group mode: back arrow + color dot + group name + 5 group nav items), Header (mobile toggle)
- **Hooks**: `src/hooks/` ŌĆö useItemMutations (14 mutations), useCalendar, useWeekCalendar, useGroupCalendar, useTodoListTasks, useCategories, useGroups

**dooooApp is the source of truth** for all business logic. Before implementing any feature in dooooWeb, always check dooooApp's equivalent screen for the correct behavior, data flow, and edge cases.

## Known Issues

### ŌÜĀ’ĖÅ OAuth buttons are non-functional

Login page renders Google and Apple OAuth buttons but they are purely visual ŌĆö clicking them does nothing. No OAuth redirect or token exchange is implemented. Email login is the only working auth method. This blocks users who signed up via OAuth on dooooApp from logging in on dooooWeb.

### ŌÜĀ’ĖÅ Display Settings page has no UI despite complete logic layer

`display-context.tsx` contains working display preference logic (date format with 3 options, time format with auto/12h/24h and per-language defaults, week start day Sunday/Monday).
